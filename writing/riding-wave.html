Lessons from Riding Waves, Applied to Life

April 2026
Fix the SHA mismatch error for writing/index.html. The saveIndex function needs to fetch a fresh SHA every time it retries, not just once before the retry loop.

Replace the saveIndex function with this pattern:

async function saveIndex() {
  const html = buildIndex(S.essays);
  const encoded = b64enc(html);
  const path = `${WP}/index.html`;
  
  const doSave = async (retryCount = 0) => {
    let sha = null;
    try { const ex = await api(`contents/${path}`); sha = ex.sha; } catch(e) { sha = null; }
    try {
      await api(`contents/${path}`, 'PUT', {
        message: 'Update writing index',
        content: encoded,
        sha: sha || undefined
      });
    } catch(e) {
      if (e.message.includes('does not match') && retryCount < 3) {
        await new Promise(r => setTimeout(r, 300));
        return doSave(retryCount + 1);
      }
      throw e;
    }
  };
  
  await doSave();
}

The critical fix is that sha is fetched INSIDE the retry function so each retry gets a completely fresh SHA from GitHub rather than reusing a stale one.
